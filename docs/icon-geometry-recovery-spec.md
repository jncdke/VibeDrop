# App 图标几何回正规格

## 1. 背景

上一轮为了解决图标模糊问题，把默认正式母版改成了：

- `图标.jpg` 提供紫色背景
- `状态栏图标.png` 提供高清白色符号

清晰度方向是对的，但几何基准选错了。

## 2. 问题

`状态栏图标.png` 的白色符号几何比例，并不等于用户之前认可的 `图标.jpg` 成品比例。

实际测量结果：

- `图标.jpg` 白色内容包围盒宽高比约 `0.892`
- `状态栏图标.png` 白色内容包围盒宽高比约 `0.997`

这意味着：

- 高清符号原始版本更方
- 直接拿它替代旧母版，会让图标显得更矮、更扁

用户实际反馈：

- 桌面端看起来“缩窄了”
- 手机端看起来“压扁了”

## 3. 目标

本次修正目标：

1. 保留上一轮已经解决的清晰度提升
2. 恢复用户认可的旧版图标几何比例
3. 不改变当前已经确认没问题的整体风格、配色和留白方向

## 4. 正确方案

正式母版应采用：

- 旧版 `图标.jpg` 作为几何参考
- `状态栏图标.png` 作为高清边缘参考

也就是：

1. 从 `图标.jpg` 中测量白色图形的目标包围盒
2. 从 `状态栏图标.png` 中提取高清白色符号 alpha
3. 把高清符号非等比变形到旧版包围盒尺寸
4. 再贴回紫色背景

这样输出的新 `图标.png` 会满足：

- 外轮廓比例与旧版一致
- 白色曲线边缘比旧版更干净

## 5. 设计原则

这里故意允许“非等比”变形。

原因不是为了偷懒，而是为了保持品牌资产在用户心里的既有观感。

在这个场景下，用户认可的不是某个抽象几何原型，而是旧版成品图整体观感。因此：

- 几何基准应服从既有成品
- 高清素材只是用来替换边缘质量
- 不能反过来让高清素材决定最终轮廓

## 6. 验证

新母版应满足：

1. `图标.png` 的白色包围盒覆盖率接近 `图标.jpg`
2. 包围盒宽高比接近 `0.892`
3. 手机端 launcher 图标不再显得被压扁
4. 桌面端 App Icon 回到之前认可的高挑轮廓
