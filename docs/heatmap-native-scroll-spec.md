# 活跃热力图滚动交互重构规格

## 1. 现状问题

当前移动端历史页热力图采用“手写拖拽 + JS 惯性外推 + 松手后按列取整”的实现。

核心表现问题：

- 拖动时像在推一个被脚本控制的层，不像原生内容滚动。
- 松手后不是自然减速，而是根据位移和速度直接结算成“跳几天”。
- 到边界时是硬夹紧，没有原生滚动的阻尼感。
- 交互语义混杂：视觉上像时间轴，行为上却更像离散分页器。

## 2. 目标效果

重构后的热力图应具备以下手感：

1. 手指左右拖动时，内容由系统原生横向滚动驱动，跟手自然。
2. 松手后保留系统惯性和减速，而不是业务 JS 自己算 180ms 外推。
3. 停止时自动吸附到最近的“天”列，保持窗口整齐。
4. 顶部“当前窗口”文案、统计卡片、回到最近按钮，跟当前可见窗口同步。
5. 热力图格子的点击筛选继续保留，不因滚动机制重构而失效。

## 3. 技术方案

### 3.1 交互底座

改为：

- `overflow-x: auto`
- `scroll-snap-type: x proximity` 或 `mandatory`
- 原生横向滚动

不再使用：

- `pointerdown / pointermove / pointerup` 手写拖拽
- `transform: translate3d(...)` 驱动主滚动
- `velocity * 惯性毫秒` 的位移预测

### 3.2 视图模型

热力图改为一次性渲染当前过滤结果范围内的全部日期列：

- 左边界：当前结果集中最早的日期
- 右边界：当前过滤语义下的最近日期
- 视口宽度：固定展示 5 列
- 滚动位置：决定“当前窗口 5 天”

### 3.3 窗口同步

通过 `scrollLeft` 和实际列步长推导当前窗口：

- `startIndex = round(scrollLeft / actualColumnStep)`
- `visibleStart = minDate + startIndex`
- `visibleEnd = visibleStart + 4`

然后同步更新：

- 顶部说明文案
- 统计卡片
- “回到最近”按钮显隐
- 内部 `viewportEndDate`

### 3.4 列步长来源

不再依赖硬编码步长常量作为真实滚动依据。

改为优先使用 DOM 实测：

- 第二列 `offsetLeft - 第一列 offsetLeft`

只有在列数不足时才退回 CSS 变量计算值。

### 3.5 视口壳层与滚动层分离

热力图不能把“视觉内边距”和“真实滚动视口”放在同一个元素上。

原因是：

- 如果横向滚动容器本身带左右 `padding`，左侧 padding 会表现成留白。
- 但右侧 padding 不会阻止后续内容继续进入可视区。
- 结果就是默认停在第 1 列时，左侧看起来正常，右侧却会漏出第 6 列的一小截。

因此结构应拆成两层：

- 外层壳：负责圆角、边框、背景、阴影、视觉内边距。
- 内层滚动区：负责 `overflow-x: auto`、`scroll-snap`，宽度严格等于 `可见列宽总和`。

移动端默认可见宽度必须精确满足：

- `5 * columnWidth + 4 * columnGap`

桌面独立窗口默认可见宽度必须精确满足：

- `7 * columnWidth + 6 * columnGap`

任何额外视觉留白，都只能加在外层壳，不能直接加在滚动视口上。

## 4. 预期收益

1. Android WebView 会直接使用系统级滚动惯性，手感明显自然。
2. 逻辑从“模拟滚动”变成“真实滚动 + 状态同步”，复杂度更低。
3. 后续如果视觉尺寸调整，不容易再出现手势数学和 CSS 尺寸漂移。

## 5. 桌面独立窗口补齐方案

在移动端和共享前端验证通过后，桌面独立窗口也要切到同一套机制，避免出现：

- 手机和桌面浏览器是原生滚动，桌面 App 还是脚本拖拽。
- 同一个热力图组件在不同入口出现不同惯性、不同边界手感、不同窗口同步时机。

桌面独立窗口的差异点只有两处：

1. 视口宽度固定展示 7 列，而不是移动端的 5 列。
2. 样式变量使用 `--desktop-heatmap-*`，列步长回退值也要从桌面变量读取。

除此之外，状态机原则保持一致：

- 不再保留 `dragPointerId / dragOffsetX / dragVelocity` 这类手写拖拽状态。
- 改为记录当前已渲染的数据、计数映射、边界信息和滚动帧句柄。
- 通过 `scrollLeft / actualColumnStep` 推导当前窗口的起止日期。
- 重置按钮直接把滚动窗口回到最近日期，再重渲染同步统计和说明文案。
