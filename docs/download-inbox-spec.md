# VibeDrop 专属收件箱目录规格

## 1. 背景

当前手机发送到 Mac 的文件最终直接落在 `~/Downloads` 根目录。

这个做法的优点是简单、符合系统默认下载语义，但在高频传输场景下有明显问题：

- 浏览器下载、聊天工具导出、系统下载和 VibeDrop 收到的文件混在一起
- 用户后续批量整理、回看、删除时定位成本高
- 产品缺少“这是 VibeDrop 收到的内容”的独立空间

因此需要把最终交付目录升级为一个**用户可见、语义清晰、自动维护**的专属收件箱。

## 2. 目标

本次目标：

1. 手机发送到 Mac 的普通文件统一保存到 `~/Downloads/VibeDrop 收件箱`
2. 如果目录不存在则自动创建，存在则复用
3. 不改变现有重名避让、历史记录、临时文件清理机制
4. 前端提示和历史文案同步更新为“VibeDrop 收件箱”

本次不做：

- 按日期或设备再分层子目录
- 用户可配置自定义保存目录
- Finder 快捷跳转或“打开收件箱”按钮

## 3. 设计原则

### 3.1 用户内容与程序状态分离

`~/.vibedrop` 适合放：

- 历史日志
- PIN
- 临时分块文件
- 签名材料

但它不适合放最终交付给用户的文件，因为它是状态目录，不是内容目录。

最终文件应当位于 Finder 容易发现、用户有心理预期的位置。

### 3.2 保留系统语义，同时增加产品语义

直接保存到 `~/Downloads` 只有系统语义，没有产品语义。

保存到 `~/Downloads/VibeDrop 收件箱` 可以同时满足：

- 仍然属于“下载内容”
- 但在 Finder 中有明确的 App 专属归属

这是比“直接扔进下载根目录”更专业的方案。

### 3.3 临时接收与最终交付分层

分块传输期间的 `.part` 和元数据文件继续放在：

- `~/.vibedrop/incoming-download-temp`

最终成功文件再移动到：

- `~/Downloads/VibeDrop 收件箱`

这样失败清理和用户可见内容不会互相污染。

## 4. 需求规格

### 4.1 功能需求

1. 文件接收完成后，默认落到 `~/Downloads/VibeDrop 收件箱`
2. 旧的单包 `file_download` 兼容路径也必须落到同一目录
3. 历史记录中的 `savedPath` 指向新目录
4. 发送成功 toast 与历史提示文案统一为“已保存到 VibeDrop 收件箱”

### 4.2 正确性需求

1. 目录不存在时自动创建
2. 同名文件继续沿用现有 `unique_path(...)` 逻辑
3. 目录创建失败时给出明确错误
4. 分块传输失败时不能在收件箱留下半成品

### 4.3 体验需求

1. 用户不需要手动设置目录
2. Finder 中能直观看出该目录是 VibeDrop 的专属收件箱
3. 文案避免再用模糊的“下载文件夹”表述

## 5. 实施方案

### 5.1 桌面端落盘目录

在桌面端新增专属目录函数：

- 下载根目录：`~/Downloads`
- 最终收件箱目录：`~/Downloads/VibeDrop 收件箱`

所有“手机 -> Mac 文件”落盘逻辑统一改用该目录。

### 5.2 保持兼容

当前仍保留旧协议 `file_download` 的兼容处理，因此旧路径也必须同步修改，避免出现：

- 新分块传输进收件箱
- 旧兼容传输还进下载根目录

这种行为分裂。

### 5.3 文案同步

以下用户可见文案同步改为：

- `文件已保存到 VibeDrop 收件箱`
- `已保存到 VibeDrop 收件箱`

## 6. 后续演进

如果后面出现更强的批量导入需求，第二阶段可继续扩展为：

- `~/Downloads/VibeDrop 收件箱/2026-04-15/`
- `~/Downloads/VibeDrop 收件箱/来自 MacBook Air/`
- `~/Downloads/VibeDrop 收件箱/2026-04-15 来自 xxx/`

但第一阶段先把“专属收件箱”建立起来最重要。
