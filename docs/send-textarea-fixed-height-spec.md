# 发送输入框固定高度规格

## 背景

Android 端发送页的每个设备卡片都会渲染一个 `textarea` 作为文字输入框。现有样式只设置了 `min-height: 116px`，同时允许 `resize: vertical`，因此某个输入框在当前 WebView 生命周期内可能被单独拉高，导致同一页面中两个设备卡片的输入框高度不一致。退出 APP 后 DOM 重建，临时高度状态丢失，所以高度又恢复一致。

## 目标

发送页设备卡片里的文字输入框必须始终保持固定高度。连接、断开、聚焦、失焦、输入长文本、页面重渲染或用户触摸输入框右下角时，都不能改变输入框外部高度。

## 非目标

不改设置页、历史筛选页等其他表单控件的布局；不改变发送按钮、设备卡片、连接状态逻辑；不改变草稿保存和焦点恢复逻辑。

## 方案

在 `.input-group textarea` 上同时设置固定 `height`、`min-height`、`max-height`，并把 `resize` 从 `vertical` 改为 `none`。长文本使用 `overflow-y: auto` 在输入框内部滚动，避免内容撑高输入框或挤压下方按钮。

## 验收

1. 默认渲染时，多个设备卡片的发送输入框高度一致。
2. 聚焦第一个输入框后，高度不变。
3. 输入大量多行文本后，输入框外部高度不变，内容可在框内滚动。
4. 连接状态从断开切到已连接，或从已连接切回断开，输入框高度不变。
5. 浏览器/WebView 不再显示可拖拽改变 textarea 高度的交互。
