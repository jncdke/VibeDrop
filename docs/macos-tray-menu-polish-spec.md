# macOS 菜单栏菜单进化规格

## 1. 背景

当前菜单栏图标尺寸已经修正，但展开后的菜单仍然偏“工程态”：

- 顶部使用 `VibeDrop - 运行中`
- 信息行使用 emoji：`📡`、`🔑`
- 主要展示的是底层运行信息，而不是面向长期使用的产品菜单结构

这会带来两个问题：

1. 视觉语气不像成熟的 macOS 常驻菜单应用
2. 关键信息可读，但可操作性不足，用户看到地址和 PIN 却不能直接复制

## 2. 目标

本次把菜单栏菜单进化到更接近正式产品的状态：

1. 文案更克制，去掉 emoji
2. 信息结构更清楚，减少“调试面板”感
3. 增加高频动作：复制地址、复制 PIN
4. 保持原生 macOS 菜单风格，不做自定义 UI

## 3. 边界

当前使用的是 Tauri 的原生托盘菜单：

- `TrayIconBuilder`
- `Menu`
- `MenuItem`
- `icon_as_template(true)`

因此这次能优化的是：

- 菜单项结构
- 文案
- 动作
- 菜单栏模板图标

不能直接优化的是：

- 菜单背景、圆角、阴影
- 字体大小
- 菜单行高
- 顶部信息区的富文本排版

如果以后要继续“进化”为更像 Raycast / Notion Calendar 那类带自定义排版的弹出层，就需要改成自定义 `popover / panel` 路线，而不是继续使用原生菜单。

## 4. 菜单结构

调整后的菜单结构：

1. `VibeDrop`
2. `地址 <ip>:<port>`
3. `PIN <pin>`
4. 分隔线
5. `复制地址`
6. `复制 PIN`
7. 分隔线
8. `打开 VibeDrop`
9. `退出 VibeDrop`

设计理由：

- 标题只保留品牌名，不再夹杂状态词
- 地址和 PIN 保持独立行，方便扫读
- 把查看信息和执行动作分开，结构更像系统菜单
- 去掉 emoji，避免和 macOS 菜单审美冲突

## 5. 动作设计

新增两个动作：

- `复制地址`
- `复制 PIN`

实现机制：

- 复用当前桌面端已接入的 `arboard` 剪贴板能力
- 菜单点击后直接写入系统剪贴板
- 失败只记录日志，不弹干扰式提示

原因：

- 这是菜单栏场景下最自然的高频动作
- 原生菜单适合“轻动作”，不适合复杂交互反馈

## 6. 不做的事

本次明确不做：

- 不自定义菜单背景与样式
- 不引入自定义浮层窗口
- 不增加菜单项图标或 emoji
- 不修改主窗口 UI
- 不修改手机端图标与菜单

## 7. 验证

1. `cargo fmt --manifest-path desktop/src-tauri/Cargo.toml`
2. `cargo check --manifest-path desktop/src-tauri/Cargo.toml`
3. `./scripts/deploy-desktop.sh --no-open`
4. 验证菜单栏展开后：
   - 信息更克制
   - 地址和 PIN 可直接复制
   - 打开主窗口和退出行为正常
