# App 图标清晰度修正规格

## 1. 问题

当前手机端启动图标的比例已经正确，但清晰度仍然偏软，尤其在 Android 桌面上会感觉白色线条边缘不够锐利。

调查结果：

- 当前正式默认源图仍然是根目录 `图标.jpg`
- 该文件只有 `784x783`
- 格式是 `JPEG`
- 图标生成链路会先把它放大到更高尺寸，再导出多档 PNG / launcher 图标

这会产生两个清晰度问题：

1. JPEG 本身是有损压缩，白色曲线边缘已经被软化
2. 低分辨率 JPEG 再被放大和多次缩放，边缘会继续变糊

## 2. 目标

本次目标不是修改图标风格，而是修正图标母版质量：

1. 保持当前紫底白图标、无金属质感的视觉方向
2. 保持当前已经确认正确的比例与留白
3. 提升手机端 launcher 图标边缘锐度
4. 同时让桌面端 App Icon 也回到无损母版链路

## 3. 设计原则

不能用“锐化 JPEG”当正式方案。

原因：

- 锐化只能增强边缘对比，不能恢复被 JPEG 压掉的真实几何边界
- 过度锐化会让白色线条出现脏边或光晕

正确方案应该是直接重建一张无损正式母版。

但要注意：

- 高清母版不能只追求边缘清晰
- 还必须保留旧版用户认可的几何比例

更完整的几何修正见 `docs/icon-geometry-recovery-spec.md`。

## 4. 方案

使用两份现有素材重建默认 App Icon 母版：

- 颜色参考：`图标.jpg`
- 高清符号参考：`状态栏图标.png`

生成规则：

1. 从 `图标.jpg` 角落采样紫色背景
2. 新建 `1024x1024` 纯紫底画布
3. 从 `状态栏图标.png` 提取白色符号 alpha
4. 把白色符号按旧版几何包围盒回正后，再以无损方式贴回紫底画布
5. 输出根目录 `图标.png` 作为高清正式母版

这样得到的结果：

- 风格仍然是你当前认可的紫底白图标
- 白色符号来自 `1024` 级高清源，不再带 JPEG 边缘噪声
- 后续 Android / macOS / Web 图标导出都基于无损母版进行

## 5. 默认源策略

默认优先级调整为：

1. 环境变量 `APP_ICON_SOURCE`
2. 自动合成并输出的 `图标.png`
3. 兼容回退：`图标_v2.png`
4. 兼容回退：`状态栏图标.png`

也就是说，`图标.jpg` 不再直接作为最终导出源，而是退化为“颜色参考素材”。

## 6. 实施范围

本次涉及：

- `scripts/generate-app-icons.py`
- 根目录高清母版 `图标.png`
- Android launcher 图标输出
- 桌面端 App Icon 输出
- 相关文档说明

不涉及：

- 菜单栏模板图标 `tray-icon.png`
- 热力图、历史页等 UI 功能

## 7. 验证

1. `python3 scripts/generate-app-icons.py`
2. `python3 -m py_compile scripts/generate-app-icons.py`
3. 检查根目录 `图标.png` 是否生成
4. 对比 `图标.png` 与 `图标.jpg`，确认风格一致但边缘更干净
5. `./scripts/deploy-android.sh --device <serial>`
6. `./scripts/deploy-desktop.sh --no-open`
7. 在手机桌面观察图标边缘清晰度是否明显提升
