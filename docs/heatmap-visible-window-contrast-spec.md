# 活跃热力图窗口对比度优化规格

## 背景

历史页活跃热力图一次只展示当前窗口内的若干日期，但旧颜色算法使用全部筛选结果里的全局最高小时计数作为颜色最大值。这样当历史范围很长、某个远处日期峰值很高时，当前窗口内的格子会被整体压浅，导致用户在当前 5 天窗口中很难判断相对高低。

## 目标

热力图颜色只表达当前可见窗口内的相对活跃度。用户左右滑动窗口后，颜色应跟随新窗口重新归一化。色阶需要显著增强，从低到高为白色、绿色、黑色，让最高峰值有明确视觉锚点。

## 行为规格

1. 当前窗口归一化：
   - 颜色最大值只取当前可见窗口内所有日期、所有小时格子的最大计数。
   - 不再使用全部历史或全部筛选范围的最大计数。

2. 滑动更新：
   - 初次渲染时按初始可见窗口计算颜色。
   - 用户左右滑动热力图后，窗口统计和格子颜色一起更新。

3. 色阶：
   - 0 条或无数据为白色。
   - 中间强度为绿色。
   - 当前窗口最大值为黑色。
   - 中低值也要有可见差异，避免大片浅灰/浅绿难以比较。

4. 非目标：
   - 不改变热力图格子数量、布局、滑动方式。
   - 不改变点击小时格筛选历史记录的行为。
   - 不改变统计卡片口径，仍统计当前可见窗口。

## 技术方案

新增当前窗口最大值计算函数，按 `visibleStart` 到 `visibleEnd` 遍历每天 24 小时，得到当前窗口 `maxCount`。渲染和滑动同步时都调用同一套颜色更新函数。颜色函数改为分段 RGB 插值：`#ffffff -> #22c55e -> #000000`，并对非零值做轻微提升，保证低值不会完全贴近白色。
